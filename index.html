TE Creation Rules

Create a TE only when review, action, or follow-up is required. Use the main idea only when naming (example: Prior Auth).

TE creation is reserved for: PA’s, Hospital/ER, Medication Clarifications, Referrals (action needed), and DME.

Note: Orders, forms, and documents requiring provider signature no longer get a TE.

Delete Only (NO TE)

These items are informational only. Do not label, file, or create a TE. Delete.

Rx / Pharmacy

  • Rx Requests (general) – Delete for active Optima providers.
  • Offboarded provider Rx requests – Create TE and send to the new Optima provider within Optima.
  • Temporary Rx Supplies – Delete.
  • Medication Non-Adherence Notices – Delete.
  • Tango approvals – Delete.

Prior Auth

  • All PA requests with a Key listed – Delete.
  • Auth notifications requested by outside provider – Delete.

Referrals

  • Confirmations of receipt – Delete.
  • Scheduled date notifications – Delete.

Imaging

  • Failure to schedule (Imaging) – Delete for now (will change once ticker workflow is aligned).

Hospital / ER TE REQUIRED

All Hospital/ER discharge summaries, notes, labs, etc. are sent to Clinical Admin.
  • If an appointment is needed, Clinical Admin completes outreach to schedule.
  • If no appointment is needed, Clinical Admin documents that and sends to provider for review.
  • Discharge summaries must be marked HP.
Document Type Label Format File To Send To Notes / Actions
Hospital / ER Records (all types) DATE – Hospital/ER – WHERE Admit / Discharge Notifications CA Bucket Route to correct CA bucket. If discharge summary, mark HP.

Lab Documents

Results attachment rule: All results ordered by Optima providers must be attached to the order.
If the order is already electronically attached, the faxed result is a duplicate and should be deleted.
Document Type Label Format File To Send To Notes / Actions
Test in Question / Coding update request DATE – Test in Question – WHERE Lab Documents Office Manager Mark HP. Note: these can also be updated on the portal.
